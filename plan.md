# plan.md

Пошаговый план реализации MVP для Cursor. Каждый шаг должен завершаться проверяемым результатом. Переходить к следующему шагу только после успешной локальной проверки текущего.

---

## Общие правила выполнения

1. Каждый шаг делай в отдельном коммите.
2. После каждого шага запускай проект и проверяй критерии готовности.
3. Не смешивай несколько больших задач в одном шаге.
4. Если какой-то шаг требует заглушек, явно отмечай TODO.
5. Сначала добиться сквозного каркаса приложения, потом подключать AI и сложную обработку.
6. Внутренний источник истины — структурированные данные в БД/JSON, не DOCX.
7. DOCX — только экспорт.
8. Все длинные операции должны возвращать понятный статус.

# Минимальный Definition of Done для MVP

MVP готов, если:
- видео загружается;
- статусы отображаются;
- аудио извлекается;
- кадры извлекаются;
- OCR работает;
- транскрибация работает;
- raw и clean transcript редактируются;
- summary / manual guide / checklist генерируются;
- knowledge block собирается;
- diff показывается;
- база знаний пополняется;
- retrieval возвращает релевантные chunks;
- чат отвечает с источниками;
- DOCX экспортируется;
- данные можно удалить.

# Рекомендуемый порядок работы в Cursor

1. Выполнять строго по шагам.
